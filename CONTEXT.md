# SongWorkbench domain context

## Product boundary

SongWorkbench is a local-first macOS application for turning a source
recording into editable practice and song-analysis assets. The application must
remain useful without network access after optional model artifacts have been
installed. Generated analysis is a draft until the user reviews it.

Source recordings and downloaded model artifacts are not repository content.

## Glossary

### Recording

An imported MP3, M4A, WAV, AIFF, or FLAC file. A recording is the immutable
source for playback, export, hashing, caching, and analysis.

### Song

The durable project identity associated with a recording. A song owns practice
settings and its latest persisted analysis document.

### Practice settings

User-controlled pitch, tempo, and loop values. These affect playback and audio
export but do not rewrite the source recording or analysis timestamps.

### Analysis run

One cancellable orchestration of selected analysis stages for a song. A run
reports stable stage progress and may preserve valid partial results when a
later independent stage fails.

### Model artifact

A versioned, downloaded ML model plus integrity digest, size, license, and
attribution metadata. An artifact is installed separately from the application
and can be verified or removed by the user.

### Stem set

Duration-aligned audio files named vocals, drums, bass, guitar, piano, and
other. A stem set can be generated by separation or imported manually, then
auditioned, mixed, and exported.

### Timed lyrics

Editable lyric segments with start/end timestamps. Machine transcription
produces a draft with confidence/provenance; manual editing produces reviewed
content only when the user explicitly marks it reviewed.

### Chord timeline

Editable timestamped chord observations plus an estimated tempo. Machine chord
classification produces confidence-bearing draft events.

### ChordPro draft

A generated or imported ChordPro document. Generation aligns the chord timeline
with timed lyrics while preserving metadata and marking uncertain content for
review.

### Analysis document

The persisted per-song aggregate containing timed lyrics, chord timeline,
ChordPro source, tempo, stem references, mixer state, provenance, and review
state. Schema migration must preserve older projects.

### Draft and reviewed

`draft` means machine-produced or changed since the last user review.
`reviewed` means the user explicitly accepted the current content. Successful
processing alone never implies review.

## Invariants

- Source recordings are never modified by analysis.
- Every cached result is keyed by source content and engine/model/schema identity.
- Cancellation stops new work and does not publish incomplete files as complete.
- A failed stage cannot erase previously persisted valid results.
- Model licenses, versions, sizes, and integrity status are visible before use.
- Lyrics and chords remain editable regardless of model availability.
- ChordPro generation is deterministic for the same reviewed inputs and options.
- UI progress begins when planning or model preparation visibly starts.
